### Продукт

- Дашборд `/start` (сводка подписок / сегодня / долги)
- Reply-навигация (Главная, Подписки, Списания, Долги, Платёж, Подписка, Настройки)
- FSM добавления подписки + карточка + list/edit/pause/delete
- Способы оплаты (CRUD-lite из FSM и настроек)
- Upcoming 7/30/all с группировкой и FX-оценкой
- **Разовый платёж (новый мастер):** название → валюта (флаги) → сумма → дата (Сегодня/Вчера/календарь) → карта → участники → деление → подтверждение; курс всегда официальный; категория = «Другое»
- **После сохранения платежа:** долги по людям + кнопки «Открыть долги / Открыть платёж / Изменить / Главное меню» (без отдельного «Меню:»)
- **Долги «кто мне должен»:** статусы 🟡 ожидает / 🟠 проверить перевод / 🟢 оплачено
- **Двухэтапное подтверждение оплаты:** deep-link `?start=debt_<token>` → друг «Я оплатил» → владелец подтверждает / отклоняет / проверить позже (24ч)
- Автонапоминание владельцу, если статус «нужно проверить» > суток
- Settings: timezone, reminder time, methods, friends, double wipe
- CBR provider + converter + DB cache
- Reminder job + idempotency keys
- **Подтверждение списания (упрощённое):** ✅ Подтвердить / ⚙️ Возникла проблема → сразу сохранить и показать следующее списание; без ввода фактической суммы
- Сценарии проблемы: не хватило денег → новая дата; дата/цена/отмена/удаление
- Команды и reply-меню всегда важнее FSM
- Редактируемая карточка списания (handlers/charges) сохранена для правок истории

## Известные ошибки / риски

1. **Секрет:** `.env.example` очищен (`BOT_TOKEN=`). Если старый токен светился в git/history — **ротировать в BotFather**.
2. FSM в MemoryStorage — обрыв после рестарта.
3. `create_all` + Alembic — нужна миграция `a1b2c3d4e5f6` на существующих БД (`alembic upgrade head`).
4. Друг взаимодействует только по deep-link; без ссылки владелец может сам нажать «Деньги пришли».
5. Формулировки «получила / подтвердила» пока женский род (имена владельца).

## Навигация и FSM

Команды и reply-меню сбрасывают FSM (см. DECISIONS D21). Deep-link `/start debt_<token>` открывает экран друга.

## Спорные решения (DECISIONS)

- D23: упрощённое подтверждение списания без факта ₽ (замена D20 ask-first).
- D24: deep-link для подтверждения долга другом.
- D25: one-time всегда CBR; категория OTHER по умолчанию.

## Что проверить вручную

1. `/start` — дашборд.
2. Напоминание → Подтвердить → «Следующее списание …».
3. Возникла проблема → каждый из 5 сценариев.
4. Разовый платёж: флаги валют → сумма с названием/валютой → Сегодня/календарь → success без «Создано долгов: N».
5. Долг → ссылка для друга → `/start debt_…` → Я оплатил → уведомление владельцу → Деньги пришли / не пришли / позже.
6. `/cancel` и Nav из середины FSM.
