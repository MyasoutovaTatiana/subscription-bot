# PRODUCT_RULES

Продуктовые и доменные правила. Где реализация уже есть — опора на код; где сценарий из ТЗ не доведён — явно помечено.

## 1. Подписки

### Создание

FSM: название → категория → сумма → валюта → периодичность → дата → способ оплаты → напоминания → друзья (сейчас skip) → подтверждение.

Категории: ИИ и работа; Видео и музыка; Образование; Вакансии; Облако и сервисы; Путешествия; Другое.

Валюты MVP: RUB, USD, EUR, CNY, JPY, VND, THB, TRY, GEL, KZT, AMD, AED.

Периодичность (`BillingType`):


| Тип                       | Правило следующей даты после списания                                     |
| ------------------------- | ------------------------------------------------------------------------- |
| `monthly`                 | +1 месяц, день `billing_day`, clamp до последнего дня месяца (31 → 28/30) |
| `every_n_days` / `custom` | +N дней (`billing_interval` ≥ 1)                                          |
| `yearly`                  | +1 год; 29.02 → 28.02 в невисокосный                                      |
| `none`                    | следующего списания нет (`next_charge_date = NULL`)                       |


Статусы отображения (UI, не отдельное поле БД):

- 🟢 Активна — `is_active` и дата не в прошлом / не «скоро»
- 🟡 Скоро списание — активна и до даты ≤ 3 дней
- 🔴 Просрочена — активна и `next_charge_date < today`
- ⏸ Приостановлена — `is_active = false`

Действия карточки: изменить поля, приостановить / возобновить, удалить (с подтверждением).

### Участники подписки

Модель `SubscriptionParticipant` есть. В FSM добавления шаг друзей **не выбирает друзей** (кнопки «без друзей» / «позже»). Долги по подписке создаются при подтверждении списания, если участники уже есть в БД.

## 2. Валюты и конвертация

- Все суммы и курсы — `Decimal`; в БД — `Numeric`.
- RUB → RUB: курс 1, округление итоговой суммы до копеек.
- Иначе: `CurrencyConverter` → провайдер `ExchangeRateProvider` (реализация CBR).
- Курс на дату: только `rate_date ≤ on_date` (не из будущего).
- Нет курса на дату → последний доступный ≤ даты (кэш / предыдущие дни CBR).
- CBR недоступен → кэш; бот не падает.
- Промежуточные значения по возможности не округлять; **итог в ₽** — `quantize` до 0.01.
- **Отображение курса:** max 2 знака — `1 USD = 76,62 ₽`.
- **Отображение денег:** max 2 знака, без лишних нулей.

Разовый платёж при создании: всегда официальный курс на дату (`cbr`). Ручной курс / факт ₽ — в сервисе, не в мастере создания (D25).

## 3. Логика списаний

Подтверждение из напоминания:

1. Кнопки: **Подтвердить списание** / **Возникла проблема**.
2. Подтвердить → сразу `confirm_charged(actual_rub=None)` → «Списание подтверждено» + следующая дата.
3. Возникла проблема → не хватило денег / дата / стоимость / отменена / удалить.
4. «Не хватило денег» и «Изменилась дата» → ввод новой даты.
5. Редактирование сохранённого списания — `handlers/charges` (история).

Разовый платёж **не** создаёт `next_charge_date` и reminders.

Ближайшие списания: группировка по дате; итоги по валютам отдельно + ориентировочный общий ₽.

## 4. Долги

### «Мне должны» (реализовано)

- Долг на друга при разовом платеже с делением или charge подписки с participants.
- Статусы: `active` 🟡 → `needs_review` 🟠 → `paid` 🟢 / `cancelled`.
- Deep-link `?start=debt_<token>`: друг «Я оплатил» → владелец подтверждает / отклоняет / «проверить позже» (24ч).
- Автонапоминание, если needs_review > суток.
- Владелец может подтвердить поступление без участия друга.

### «Я должен» (не реализовано)

Нет модели обязательств владельца перед друзьями.

### Деление

Сервис: equal / percent / fixed / custom_shares.  
**UI разового платежа:** equal или «без деления».  
Остаток копеек → владельцу (если в сплите), иначе последнему участнику.

## 5. Уведомления

- Defaults: TZ `Europe/Moscow`, время `10:00`, offsets `(3, 1, 0)`.
- Jobs: напоминания подписок + debt review.
- Кнопки напоминания: Подтвердить списание, Возникла проблема.

## 6. Подтверждение оплаты другим человеком

Реализовано через deep-link (D24). Друг не закрывает долг самостоятельно.

## 7. Перенос и пропуск


| Действие            | Поведение                                      |
| ------------------- | ---------------------------------------------- |
| Новая дата          | Через «Возникла проблема» или edit подписки    |
| Смена суммы         | Через проблему / edit                          |
| Приостановка        | `is_active=false`                              |


## 8. Отображение сумм и дат

- Даты: `20 июля 2026` / кратко `14 августа`.
- Деньги: `1 500 ₽`, `1 500,50 ₽`, `$20,40`.
- Курс: `1 USD = 76,62 ₽`.
- Без tech jargon в UI.
- Подписки: мягкий hint без акцента на «ЦБ».
- Подтверждение разового платежа: допускается «Курс ЦБ» + `RUB_HINT_CBR`.
